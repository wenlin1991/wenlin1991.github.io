Introduction to Java Threads

Theads are the most basic way of obtaining concurrency in Java

  • A Java thread is a unit of computation that runs in the context of a process.
  • A process is a unit of resource allocation & protection.
  • Java enables multiple threads to run in multiple processes.
    • One thread can have multiple threads
  • Thread communicating
    • Java threads running in the same process can communicate with each other via shared objects or message passing. (Message is nothing but Data Structure)
    • Java threads running in different processes can communicate with each other via shared memory or inter-process communication (IPC) mechanisms. (IPC: Sockets) (Shared Memory is not vanilla Java, but it supported by a bunch of other frameworks, like Android. )
  • Thread unique states and common states
    • Each Java thread leverages unique “state” from the underlying Operating System thread, e.g., a runtime stack, an instruction counter, & other registers
    • Java dynamic & static objects can be shared across Java threads. (That’s why we can pass message around.)

Giving Code to a Java Thread

  • Java threads must be given code to run

    Do not use the “no argument” Thread constructor directly!!!

  • Extend the Thread class

    Override the run() hook method in the subclass & define the thread’s computations.

    Create & start a thread using a named subclass of Thread.

    • Pros
      • It’s straightforward to extend the Thread super class
        • Just override the run() hook method!
      • It also consolidates all state & methods in one place.
        • Enables central allocation & management of the thread
        • This design is useful when the thread must be updated during runtime configuration changes
    • Cons
      • A subclass must extend the Thread superclass
        • This is restrictive since Java only allows one superclass per subclass

  • Implementat the Runnable interface

    Implement the run() hook method of an interface to define the thread’s computations.

      Create & start a thread using an anonymous inner class as the runnable.

      A lambda expression is an unwieldy if the code to run is long, complex, or needs to be used multiple times.

Passing Parameters to a Java Thread

Parameters passed to run() can be supplied via one of two other means.

  • As parameters to a class constructor

    Use the field(s) within the thread’s run() hook method to customize its behavior.

  • As parameters to “setter” methods

Running Java Threads

  • There are multiple layers involved in creating & starting a thread
    • Creating a new thread object doesn’t allocate a run-time call stack of activation records.
    • The runtime stack & other thread resources are only allocated after the start() method is called.
    • The Java execution environment calls a thread’s run() hook method after start() creates its resources.
    • Each thread can run concurrently & block independently
  • Any code can generally run in a thread
    • However, windowing toolkits often restrict which thread can access GUI components.
      • e.g., Only the Android UI thread can access GUI components

Life cycle of a thread

  • A thread can live as long as its run() hook method hasn’t returned.

    • The underlying thread scheduler can suspend & resume a thread many times during its lifecycle.
      • Scheduler operations are largely invisible to user code, as long as synchronization is performed properly.

  • For a thread to execute “forever”, its run() hook method needs an infinite loop.

  • The thread is dead after run() returns.

    • A thread can end normally

    • Or an uncaught exception can be thrown

  • The join() method allows one thread to wait for another thread to complete

    • A thread can simply evaporate! (蒸发，消失)
    • The Java execution environment recycles thread resources
      • e.g., runtime stack of activation records, thread-specific storage, etc.

Some Common Java Thread Methods

  • There are a number of commonly used methods in the Java Thread class, e.g.,
    • void setDaemon()
      • Marks thread as a daemon
      • What is a daemon thread ?
    • void start()
      • Allocates thread resources & initiates thread execution by calling the run() hook method
    • void run()
      • Hook method where user code is supplied
    • void join()
      • Waits for a thread to finish
    • void sleep(long time)
      • Sleeps for given time in ms
    • Thread currentThread()
      • Object for current Thread
    • void interrupt()
      • Post an interrupt request to a Thread
    • boolean isInterrupted()
      • Tests whether a thread has been interrupted
    • boolean interrupted()
      • Tests whether current thread has been interrupted
    • void setPriority(int newPriority) & int getPriority()
      • Set & get the priority of a Thread

Java Thread Happens-Before Orderings

  • Java Threads methods established happens-before orderings

    • Ensure that if one event happens-before another event, the result must reflect that, even if those events are actually executed out of order.
    • In general, a happens-before relationship guarantees that memory written to by statement A is visible to statement B

  • Examples of happens-before orderings in Java

    • Starting a thread happens-beofre the run() hook method of the thread is called

      The lambda () -> System.out.println("hello world") plays the role of the run() hook method.

      A thread’s state is consistent & visible before run() starts.

    • Methods in java.util.concurrent package classes also establish happens-before orderings.

    • The termination of a thead happens-before a join() with the terminated thread

      A thread waiting on a (non-timed) join() only resumes after the target thread terminates
